Adding an Alt Tag to an Image in Classic Form Builder
To add an alt tag to an image in Classic Form Builder, locate the Element that contains the image and open its Properties. Within the properties, add an alt attribute with a descriptive value to support screen readers and improve accessibility.
Example: alt = 'this is an alt tag for screen readers'
Ensure the alt text clearly describes the image’s purpose or content for users who rely on assistive technologies.

Heading Hierarchy: Headings must follow a structured order (H1 for the form title, H2s for sections, H3s for subsections) to support screen reader navigation and content understanding. Font sizes should align with this hierarchy so that larger headings indicate higher-level sections and smaller headings indicate subsections. Using proper heading levels allows assistive technologies to identify sections correctly and enables users to navigate the form efficiently.

Information

  • Screen Reader Compatibility: All content, including instructions, labels, errors, and button text, must be readable and understandable using screen readers such as NVDA (Windows) and VoiceOver (macOS). both of these tools are free to use and will validate many points listed on this form.
  • Proper Labeling of Form Fields: All input fields must have programmatically associated labels so screen readers can announce the purpose of each field clearly and accurately.
  • Required and Optional Field Identification: Required fields should be clearly indicated both visually and programmatically so assistive technologies can announce their status.

  • Keyboard Navigation: Users must be able to navigate, complete, and submit the form using the keyboard alone, without requiring mouse interaction.
  • Logical Tab Order: Keyboard focus should move through the form in a logical, predictable order that matches the visual and reading flow of the page.
  • Keyboard Accessibility: All form elements must be fully usable using a keyboard alone, without requiring a mouse or causing keyboard traps.

  • Error Identification and Messaging: Validation errors must be clearly described, associated with the correct field, and conveyed without relying on color alone.
  • Visible Focus Indicators: A clear visual indicator should show which field or element currently has keyboard focus as users tab through the form.
  • Instructions and Help Text: Instructions should appear before the fields they relate to and be programmatically connected so screen readers announce them at the appropriate time.
  • Color Contrast and Non-Color Cues: Text and interactive elements must meet minimum color contrast requirements, and color should not be the only method used to convey information.
  • Logical Grouping of Related Fields: Related fields, such as radio buttons or checkbox groups, should be grouped together to provide clear context for assistive technology users.
  • Clear Button Labels and Actions: Buttons and actions should have descriptive labels so their purpose is clearly announced by screen readers.
  • Consistent Layout and Predictable Behavior: The form layout and interactions should behave consistently to reduce confusion and support accessibility for all users.

ADA From Summary Checklist

  • ✅ A single H1 is used for the form title, with H2 and H3 headings following a logical hierarchy.
  • ✅ All form fields have clearly associated labels so their purpose is announced by screen readers.
  • ✅ Required fields are clearly identified both visually and for assistive technologies.
  • ✅ Instructions and help text appear before the fields they relate to.
  • ✅ The form can be completed using the keyboard alone, without requiring a mouse.
  • ✅ Tab order follows a logical sequence that matches the visual layout of the form.
  • ✅ A visible focus indicator clearly shows which element currently has keyboard focus.
  • ✅ Validation error messages are text-based, descriptive, and associated with the correct field.
  • ✅ All content is readable using screen readers such as NVDA (Windows) and VoiceOver (macOS).
  • ✅ Text and interactive elements meet minimum color contrast requirements.
  • ✅ Color is not the only method used to convey meaning or status.
  • ✅ Related controls, such as radio buttons and checkboxes, are logically grouped.
  • ✅ Buttons and actions use clear, descriptive labels that communicate their purpose.
  • ✅ Images include appropriate alt text, or are marked as decorative when applicable.

For detailed requirements, examples, and testing guidance, refer to the WCAG 2.1 guidelines, which are widely used to demonstrate ADA compliance: WCAG 2.1 Understanding Accessibility Guidelines This resource explains why each requirement exists, how assistive technologies interpret forms, and how to test accessibility effectively.
